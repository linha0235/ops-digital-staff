# 运维数字员工系统部署文档

## 硬件要求
PC机最低配置：4核8GB内存 100GB硬盘，可支持本地部署全部服务。

## 一、本地大模型部署（DEEPSEEK）
1. 下载 Ollama 运行时：https://ollama.com/
2. 执行命令拉取并运行DeepSeek模型：
```bash
ollama run deepseek-r1:7b
```
模型启动后本地端口11434即可对外提供大模型推理能力。

## 二、RAG知识库部署（AnythingLLM）
1. 使用Docker一键部署AnythingLLM：
```bash
docker run -d -p 3001:3001 -v anythingllm-storage:/app/server/storage mintplexlabs/anythingllm
```
2. 访问 http://localhost:3001 完成初始化配置，创建工作空间，获取API Key。
3. 修改backend/src/main/resources/application.yml中AnythingLLM配置项：
```yaml
anything-llm:
  base-url: http://localhost:3001/api
  api-key: 你自己的API Key
  default-query-workspace: 你创建的工作空间Slug
```

## 三、业务系统后端部署
1. 本地安装JDK17、Maven、MySQL8.0、Redis
2. 执行backend/src/main/resources/sql/init.sql初始化数据库
3. 修改application.yml中数据库和Redis连接地址
4. 进入backend目录执行mvn package打包，启动jar包
5. 接口文档地址：http://localhost:8080/doc.html

## 四、业务系统前端部署
1. 进入frontend目录执行：
```bash
npm install
npm run dev
```
2. 访问前端地址：http://localhost:5173
3. 初始账号admin/123456

## 完整业务流程
1. 运维人员在知识库页面新增FAQ问答对，点击同步到向量库
2. 用户在智能问答页面输入问题，系统自动调用RAG返回知识库答案
3. 遇到无法回答的问题时，用户可以提交人工报障工单
4. 后台运维人员在工单管理页面处理工单，录入处理结果
5. 工单完成后可将新的问题和解决方案补充到知识库，形成闭环
