# 运维数字员工系统部署文档

## 硬件要求
PC机最低配置：4核8GB内存 100GB硬盘，可支持本地部署全部服务。

## 一、基础中间件部署（Docker Compose）

项目根目录提供了 `docker-compose.yml`，可一键启动 MySQL、Redis、AnythingLLM：

```bash
docker-compose up -d
```

仅需 MySQL + Redis（已有 AnythingLLM 或 Ollama 时）可使用轻量编排：

```bash
docker-compose -f docker-compose-light.yml up -d
```

MySQL 初始化脚本 `backend/src/main/resources/sql/init.sql` 会自动执行，创建数据库表并插入默认管理员账号。

## 二、本地大模型部署（Ollama）

1. 下载 Ollama 运行时：https://ollama.com/
2. 执行命令拉取并运行模型：
```bash
ollama pull qwen2.5:3b
```
模型启动后本地端口 11434 即可对外提供大模型推理能力。

> 如需更换模型，修改 `AnythingLLMClient.java` 中 `callOllamaStream` 方法的 `model` 参数即可。

## 三、RAG 知识库部署（AnythingLLM）

1. 使用 Docker 一键部署（包含在 docker-compose.yml 中）：
```bash
docker run -d -p 3001:3001 \
  -v anythingllm-storage:/app/server/storage \
  -e DISABLE_TELEMETRY=true \
  mintplexlabs/anythingllm
```
2. 访问 http://localhost:3001 完成初始化配置，创建工作空间，获取 API Key。
3. 修改 `backend/src/main/resources/application.yml` 中 AnythingLLM 配置项：
```yaml
anything-llm:
  base-url: http://localhost:3001/api
  api-key: 你自己的API Key
  default-query-workspace: 你创建的工作空间Slug
```

## 四、业务系统后端部署

1. 本地安装 JDK17、Maven 3.8+、MySQL 8.0、Redis 7.x
2. 如未使用 Docker，手动执行 `backend/src/main/resources/sql/init.sql` 初始化数据库
3. 修改 `application.yml` 中数据库和 Redis 连接地址
4. 进入 backend 目录执行：
```bash
mvn clean package -DskipTests
java -jar target/digital-staff-1.0.0.jar
```
5. 接口文档地址：http://localhost:8080/api/doc.html

## 五、业务系统前端部署

### 开发模式
```bash
cd frontend
npm install
npm run dev
```
访问 http://localhost:5173，默认账号 `admin / 123456`。

Vite 已配置 `/api` 代理转发到 `http://localhost:8080`，开发时无需额外配置。

### 生产模式
```bash
cd frontend
npm run build
```
生成的 `dist/` 目录通过 Nginx 部署，`/api` 请求转发至后端 8080 端口。

Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name _;

    location / {
        root /var/www/ops-digital-staff;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 完整业务流程

1. 运维人员在知识库页面新增 FAQ 问答对，点击同步到向量库
2. 用户在智能问答页面输入问题，系统自动调用 RAG 检索向量知识库：
   - 高置信度匹配（score > 0.7）：直接返回知识库答案，无需调用大模型
   - 低置信度匹配：拼接检索到的知识库上下文，调用 Ollama 流式推理
3. 前端通过 SSE（Server-Sent Events）接收流式响应，打字机动画逐字输出
4. 遇到无法回答的问题时，用户可以提交人工报障工单
5. 后台运维人员在工单管理页面处理工单，录入处理结果
6. 工单完成后可将新的问题和解决方案补充到知识库，形成闭环

## RAG 问答技术架构

```
用户提问（前端 Chat.vue）
  │  POST /api/faq/chat/stream (SSE)
  ▼
OpsFaqController.chatStream()
  │  SseEmitter
  ▼
OpsFaqServiceImpl.chatQueryStream()
  │
  ▼
AnythingLLMClient.chatCompletionStream()
  │
  ├─ ① 调用 AnythingLLM /v1/workspace/{slug}/chat (mode=query)
  │     返回 sources[] 向量检索结果
  │
  ├─ ② extractAnswer() 解析 sources
  │     score > 0.7 且正则匹配 "答案：(.+?)" → 直接返回（秒级响应）
  │
  └─ ③ 低置信度 → 拼接上下文 → 调用 Ollama /api/chat (stream=true)
        NDJSON 逐 token 解析 → 前端逐字打字机动画
```
